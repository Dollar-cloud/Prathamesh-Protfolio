Port Guard

Problem: Scattered port data. Solution: One dashboard for monitoring, charts, and alerts.

Designed a real-time security dashboard that centralizes alerts, logs, and device status to improve visibility and reduce manual monitoring effort.

Network admins need to see port activity and threats at a glance. Port Guard gives them a security dashboard where monitoring, charts, and alerts live in one place instead of scattered terminals.

Technical Challenge & Solution

Displaying live port status and traffic data without a real backend meant building a reactive UI that updates as if data were streaming. I used a mock API with simulated polling: state updates drive re-renders, and the chart library receives clean data structures. The pattern is ready to swap in real WebSocket or REST when backend exists—no UI rewrites needed.

Tech Stack — Why These Tools

  • HTML/CSS/JS: Vanilla stack for full control and fast load. No framework overhead for a focused dashboard.
  • CSS Grid & Flexbox: Layout that stays consistent across screen sizes. Grid for the main dashboard grid, Flexbox for cards and nav.
  • Chart library: Used for time-series and status viz. Chose one with good accessibility and lightweight bundle over heavier analytics suites.
  • Mock data: JSON files and a small fetch layer simulate API. Keeps logic and UI separate for easier backend integration later.

Gaming Café Booking System

Problem: Booking slots by phone or paper. Solution: UI for slot selection, calendar, and payment flow—frontend ready for backend.

Built an intuitive booking interface that simplifies PC and console reservations, improving user clarity and reducing booking friction.

Café owners and customers need to book PC/console slots without phone calls or paper. This UI handles slot selection, calendar availability, and a payment flow demo—all frontend, ready to plug into a real backend.

Technical Challenge & Solution

Booking flows involve multiple steps: pick date → pick slot → confirm → pay. Keeping state coherent across steps without a framework was the main challenge. I used a simple state object and render functions: each step reads from state and dispatches updates. URL hash (#step-2) keeps the flow shareable and back-button friendly. No router library—just vanilla JS and clear separation between state and DOM.

Tech Stack — Why These Tools

  • Vanilla JS: No React/Vue for a linear flow. Keeps the project easy to understand and quick to load.
  • CSS custom properties: Theme tokens for colors and spacing. Makes dark/light modes and tweaks trivial.
  • Calendar UI: Custom-built for slot availability. Chose not to pull in a full date-picker library—we only needed day-level selection, so a simple grid was lighter and more controllable.
  • Form validation: Inline, before submit. Clear error messages and focus management for keyboard users.
